# Apps Directory

## 目的
本目录用于放项目的前端应用。

当前规划包括：
- `web/`
- `mobile/`

## 子目录职责

### `web/`
首发端。

未来主要承载：
- 今日三餐首页
- 历史 day plan 浏览
- meal item / recipe 详情页
- 购物清单页
- 食物库与营养知识内容页

### `mobile/`
后续端。

未来主要承载：
- 复用同一套推荐结果和数据接口
- 补充移动端交互与缓存体验
- 逐步对齐 Web 首版核心能力

## 当前状态
当前两端都还是占位目录。

原因不是前端不重要，而是项目目前优先级是：
1. 需求澄清
2. 文档完善
3. 规则收敛
4. 再进入实现

## 当前原则
- Web 优先
- Mobile 后续
- 推荐逻辑统一放后端
- 前端以展示、交互和缓存为主
