# 00 Project Overview

## 项目目标
这是一个面向普通健康饮食用户、兼顾家庭场景的智能饮食助手。

核心目标：
1. 生成家常、可执行、相对均衡的一日三餐
2. 帮助用户理解食物与营养
3. 提供菜谱步骤和购物材料
4. 支持按人数推荐更合理的多菜组合

## 项目要解决的问题
用户在日常饮食中常见的痛点不是“没有营养理论”，而是：
- 不知道今天吃什么
- 想吃得相对均衡，但不想做复杂计算
- 家里多人吃饭时，不知道该搭配几道菜
- 想要一个从推荐到买菜都能直接执行的结果

本项目要提供的不是“营养诊疗方案”，而是“日常饮食决策辅助”。

## 主要用户
### 1. 普通健康饮食用户
- 希望吃得更省心
- 希望有基本营养意识
- 不希望面对复杂专业术语

### 2. 家庭用户
- 需要按人数生成更合理的餐项组合
- 需要今天就能执行的菜谱和清单
- 更关注家常、稳定、易操作

## 首版核心体验
首版核心体验是一条清晰的用户链路：

1. 输入当前人数
2. 可选择少量当前不吃的食材
3. 获得今日早餐、午餐、晚餐推荐
4. 查看简短推荐理由
5. 查看部分 meal item 对应的 recipe
6. 自动获得今日购物清单
7. 可回看最近几天的 day plan 记录
8. 如有兴趣，可继续查看食物库和营养知识内容

## 产品形态
- Web：优先上线
- Mobile App：后续跟进
- Mini Program：未来再考虑，不在首发范围

## 核心模块
1. Meal Plan：一日三餐推荐结果
2. Meal Item：餐项，系统推荐最小单位
3. Recipe：部分 meal item 的详情层
4. Shopping List：今日买菜清单
5. Food Library：食物视角的内容模块
6. Nutrition Knowledge：营养素视角的内容模块

## 当前定位
本项目是：
- 健康饮食助手
- 日常决策辅助工具
- 内容与工具结合型产品

本项目不是：
- 医疗系统
- 疾病管理系统
- 营养师处方工具
- 社区平台
- 买菜电商平台

## 当前设计原则
- 推荐最小单位是 `meal item`
- 不是所有 `meal item` 都有完整 `recipe`
- 以“全天均衡”为主，单餐只要求基础合理
- 首版不追求医疗级营养精度
- 首版不做复杂个性化组合
- 首版优先做简单、稳定、可维护的方案
